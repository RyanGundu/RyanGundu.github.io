Geaneology API

The Genealogy API is my largest project yet; the final product is a full stack application. It was completed throughout the span of a semester. The GEDCOM file parser was made with C, the front end with HTML and CSS. Also, I implemented a Node.js server to allow communication between the front end and a MySQL DBMS.

File Type

GEDCOM

What is a GEDCOM file? To break it down, it serves as a family tree but in the form of a text file. It contains records for each individual, within a record there could be a name, date of birth etc. Family records link to multiple individual records indicating a single family. Also, each individual can be linked to multiple family records (Ex. To be a mother and a daughter). For more on GEDCOM files click here.

UI

Features

With the parser complete there is much to do with the data. The Application I designed allowed users to upload multiple GEDCOM files. The user may then:

  • View the details of all uploaded files
    • Ex. # of Individuals and Families
  • Display all individuals in a file
  • Add individuals to a selected file (writes to the file)
  • Get the descendants of an individual
  • Get the ancestors of an individual
  • Load data to MySQL DBMS
  • Organize individuals by
    • First/Last name, Sex etc.

Users are also able to make their own custom queries to the database. The data is returned and displayed in an organized table.
